Embed a local image file (screenshot, .png/.jpg/.gif, CleanShot capture, before/after, UI capture, CI failure image) into an existing GitHub pull request — either in the PR body or as a PR comment. **The GitHub API and `gh` CLI do not support image uploads**, so you cannot do this with `gh pr edit` or `gh pr comment` alone; this skill is the only way to get a local image into a PR, and it works by driving a real browser to GitHub's user-attachments uploader. Use whenever a message references both (a) a local image — by file path, filename, "screenshot", "image", "screen recording", "cleanshot", or location like Desktop/Downloads/tmp — AND (b) an existing pull request — by number (#1234), URL, branch name ("the PR for branch X"), or phrases like "the open PR" or "the PR I just opened". The verb does not matter (upload, attach, embed, add, put, post, drop, slap, stick, include, show, document, "can this go on", "get this onto"). Also use for visually documenting test runs, bug repros, UI states, or CI failures inside an existing PR. Do NOT use for: capturing or taking a new screenshot, editing/resizing images, opening a brand-new PR (use the /pr skill), pushing branches, or referencing images inside repo files like README.md.
