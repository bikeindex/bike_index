Original: Upload local images to a GitHub PR and embed them in the description or comments. Use when asked to "attach screenshots to PR", "add images to PR", "upload test results to PR", "embed screenshots in PR description", "add before/after images to PR", "attach UI screenshots", "show test results in PR", "add visual evidence to PR", or any request involving images and PRs. Always use this skill when the user wants to visually document changes in a pull request, even if they don't use the word "upload" — phrases like "put the screenshot in the PR" or "show the image in the PR" should trigger this skill.

Best: Embed a local image file (screenshot, .png/.jpg/.gif, CleanShot capture, before/after, UI capture, CI failure image) into an existing GitHub pull request — either in the PR body or as a PR comment. **The GitHub API and `gh` CLI do not support image uploads**, so you cannot do this with `gh pr edit` or `gh pr comment` alone; this skill is the only way to get a local image into a PR, and it works by driving a real browser to GitHub's user-attachments uploader. Use whenever a message references both (a) a local image — by file path, filename, "screenshot", "image", "screen recording", "cleanshot", or location like Desktop/Downloads/tmp — AND (b) an existing pull request — by number (#1234), URL, branch name ("the PR for branch X"), or phrases like "the open PR" or "the PR I just opened". The verb does not matter (upload, attach, embed, add, put, post, drop, slap, stick, include, show, document, "can this go on", "get this onto"). Also use for visually documenting test runs, bug repros, UI states, or CI failures inside an existing PR. Do NOT use for: capturing or taking a new screenshot, editing/resizing images, opening a brand-new PR (use the /pr skill), pushing branches, or referencing images inside repo files like README.md.
